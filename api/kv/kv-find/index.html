KV_FIND

Searches the values and returns keys, data or paths.

ParamTypeMeaningRequired
rspstringMust be one of: keys, kv, or pathsY
pathstringA JSON Path applied to each key's valueY
keysarrayAn array of keys. If present, only these keys are searchedN

The response depends on rsp (examples below):

"keys"​

Only keys matching the path are returned.

"paths"​

The paths to the element(s) matching the path returned. The key in which the values are found are not returned.

"kv"​

The keys and values are returned for matching paths.

Response​

KV_FIND_RSP

The tkn is always returned so this is omitted from the tables below.


keys​

ParamTypeMeaning
keysarrayArray of keys

paths​

ParamTypeMeaning
pathsarrayArray of paths

kv​

ParamTypeMeaning
keysobjectFor each key that matches the criteria there's an entry: "<keyname>":<value>

Keys​

Find if we have a Samsung device
{
  "KV_FIND":
  {
    "rsp":"keys",
    "path":"$[?(@.make == 'Samsung')]"
  }
}

Response:

{
  "KV_FIND_RSP":
  {
    "keys": ["Devices"]
  }
}
Find if we have an Apple iPhone 14
{
  "KV_FIND":
  {
    "rsp":"keys",
    "path":"$[?(@.make == 'Apple' && @.model == 'iPhone 14')]"
  }
}

Response:

{
  "KV_FIND_RSP":
  {
    "keys": ["Devices"]
  }
}
Find if we have an Apple iPhone 11
{
  "KV_FIND":
  {
    "rsp":"keys",
    "path":"$[?(@.make == 'Apple' && @.model == 'iPhone 11')]"
  }
}

Response:

Keys is empty, no iPhone 11
{
  "KV_FIND_RSP":
  {
    "keys": []
  }
}

Paths​

Only the path is returned

Find devices that have at least three sensors
{
  "KV_FIND":
  {
    "rsp":"paths",
    "path":"$[?(@.sensors.length >= 3)]"
  }
}

Response:

iPhone 14
{
  "KV_FIND_RSP":
  {
    "paths": ["$[1]"]
  }
}

KV​

For this we'll use this data:

{
  "KV_SET":
  {
    "keys":
    {
      "loginsValid":
      [
        {"timestamp":1234, "location":"London"},
        {"timestamp":1235, "location":"London"},
        {"timestamp":1236, "location":"Paris"},
        {"timestamp":1238, "location":"London"}
      ],
      "loginsFailed":
      [
        {"timestamp":1235, "location":"New York"},
        {"timestamp":1235, "location":"London"},
        {"timestamp":1237, "location":"New York"}
      ]
    }
  }
}

Get the valid logins if any are between timestamps 1234 and 1238:

{
  "KV_FIND":
  {
    "rsp":"kv",
    "keys":["loginsValid"],
    "path":"$[?(@.timestamp > 1234 && @.timestamp < 1238)]"
  }
}

Response:

{
  "KV_FIND_RSP":
  {
    "kv": {
      "loginsValid": [
        {
          "timestamp": 1234,
          "location": "London"
        },
        {
          "timestamp": 1235,
          "location": "London"
        },
        {
          "timestamp": 1236,
          "location": "Paris"
        },
        {
          "timestamp": 1238,
          "location": "London"
        }
      ]
    }
  }
}

Get the failed logins if any are from London:

Restrict to loginsFailed key
{
  "KV_FIND":
  {
    "rsp":"kv",
    "keys":["loginsFailed"],
    "path":"$[?(@.location == 'London')]"
  }
}

Response:

{
  "KV_FIND_RSP":
  {
    "kv": {
      "loginsFailed": [
        {
          "timestamp": 1235,
          "location": "New York"
        },
        {
          "timestamp": 1235,
          "location": "London"
        },
        {
          "timestamp": 1237,
          "location": "New York"
        }
      ]
    }
  }
}
